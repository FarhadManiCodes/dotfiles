# CLAUDE.md

Guidance for Claude Code when working in this dotfiles repository.

## Repository Overview

Dotfiles for **Arch Linux + Niri (Wayland)**. Data engineering / scientific computing focus.
All configs are symlinked from this repo via `install.sh`. The source of truth is always the live `~/.config/` files ‚Äî changes are synced back to dotfiles, not the other way around.

## Installation

```bash
git clone --recurse-submodules git@github.com:FarhadManiCodes/dotfiles.git ~/dotfiles
cd ~/dotfiles && bash install.sh
```

Post-install:
- Zsh plugins: `~/.config/zsh/update-plugins.sh`
- Vim plugins: `vim +PlugInstall +qall`
- Tmux plugins: `Prefix + I` inside a session
- Git identity: fill in `~/.config/git/config.local`

## Architecture

### Symlink model

Every tracked config is a symlink pointing into this repo. `install.sh` creates all symlinks. When checking whether dotfiles are in sync, use `ls -la` to confirm symlinks ‚Äî not file contents.

Not tracked (intentionally):
- `~/.config/git/config.local` ‚Äî name + email
- `~/.config/gh/hosts.yml` ‚Äî auth tokens
- `~/.config/zsh/plugins/` ‚Äî cloned by `update-plugins.sh`
- `~/.config/zsh/completions/` ‚Äî generated by `generate-completions.sh`
- `gtk-3.0/bookmarks` ‚Äî local state

### Neovim (primary editor)

Submodule at `nvim/` ‚Üí [FarhadManiCodes/nvim-config](https://github.com/FarhadManiCodes/nvim-config).
Symlinked: `~/.config/nvim` ‚Üí `dotfiles/nvim`.
To update: `cd nvim && git pull`, then commit the submodule pointer in dotfiles.

### Vim (lightweight editing)

Used for markdown, config files, quick edits. Not for development.

- **Entry**: `vim/vimrc` ‚Üí sources `~/.config/vim/config/*.vim`
- **Config modules**: `basic.vim`, `plugins.vim`, `plugins_config.vim`, `autocmds.vim`, `mappings.vim`, `python.vim`, `yaml.vim`, `json.vim`
- **Fallback**: `minimal.vim` auto-loaded when invoked as `vi` or `view`
- **Plugins** (12): auto-pairs, vim-surround, tcomment, vim-repeat, vim-unimpaired, lightline, onedark, PaperColor, vim-markdown, Goyo, Limelight, vim-tmux-navigator, rainbow_csv, vim-envx
- **Theme toggle**: `<leader>tt` ‚Äî cycles onedark ‚Üí PaperColor light ‚Üí PaperColor dark
- **Leader**: `\`

Vim config auto-reloads on save. Editing `vim/config/basic.vim` takes effect immediately.

### Zsh

- **Entry**: `~/.zshrc` + `~/.zshenv`
- **Config dir**: `~/.config/zsh/`
- **Aliases**: `zsh/aliases`
- **Helpers**: `zsh/helpers.zsh` ‚Äî `safe_source` and utility functions
- **Functions** (`zsh/functions/`):
  - `bat.zsh` ‚Äî theme-aware bat wrapper
  - `clipboard.zsh` ‚Äî Wayland file copy
  - `cpp.zsh` ‚Äî C++ build helpers (cmake/ninja/ccache)
  - `fzf.zsh` ‚Äî FZF config + `fnb` (Jupyter finder) + `fdata` (data file finder)
  - `git-enhancements.zsh` ‚Äî `gci`, `gst`, `gstds`
  - `last_working_dir.zsh` ‚Äî restore last directory
  - `pdf.zsh` ‚Äî PDF/book search with rga + fzf
  - `search.zsh` ‚Äî `ff`, `fdir`, `fgit`, `rgf`, `rgpy`, `rgcpp`
  - `virtualenv.zsh` ‚Äî full uv+direnv venv management (`vc`, `va`, `vp`, `vd`, `vl`, `vr`)
- **Plugins** (not tracked): fast-syntax-highlighting, zsh-autosuggestions, zsh-history-substring-search, forgit

### Tmux

- **Config**: `tmux/tmux.conf` ‚Äî prefix `Ctrl-a`
- **Layouts**: `tmux/layouts/cpp_layout.sh` (active)
- **Archive**: `tmux/layouts/archive/` ‚Äî old layouts kept for reference

## Modifying configs

**Add a new config to dotfiles**:
1. `git checkout -b add-<name>-config`
2. Copy live file: `cp ~/.config/<name>/file dotfiles/<name>/file`
3. Add symlink to `install.sh`
4. Symlink on current machine: `ln -sf dotfiles/<name>/file ~/.config/<name>/file`
5. Commit and merge

**Sync live changes back to dotfiles**: just `cp` the changed file ‚Äî the symlink means the dotfiles file IS the live file, so this is only needed if symlinks were bypassed.

**Never commit**:
- `git/config.local`, `gh/hosts.yml`, anything with tokens/passwords/keys
- Auto-generated files (completions, accels.scm, bookmarks)
